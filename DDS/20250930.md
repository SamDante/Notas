## Notas de la clase

Recurso web: 

* Autenticación: Se exige que el usuario se identifique antes de acceder a los recursos.
* Autorización basada en roles: Se exige que el usuario, previamente identificado, cuente con un rol en particular para acceder a los recursos.
* Autorización basada en roles y permisos: Se exige que el usuario, previamente identificado, cuente con un rol particular y con uno o varios permisos para poder acceder a los recursos.

Ejemplo de modelado de usuarios, roles y permisos

```
@startuml

class Usuario {
    - nombre: String
    - contraseña: String
    - rol: Rol
}

class Rol {
    - nombre: String
    - permisos: List<Permiso>
    + tienePermiso(usuario): Boolean
}

class Permiso {
    - nombre: String
}

Usuario -> Rol
Rol ->"*" Permiso

@enduml
```
