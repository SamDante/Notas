### Arquitectura Cliente - Servidor

El cliente siempre inicia la comunicación. El servidor sólo responde a sus peticiones.

Por lo general, el cliente se enfoca en la interfaz (son aplicativos) y el servidor en la lógica y datos.

Clasificación de Cliente - Servidor según sus responsabilidades
|  | Servidor Pasivo | Servidor Activo |
| :-: | :-: | :-: |
| Cliente Pasivo | Ambos componentes poseen baja lógica de negocio o simplemente son considerados "componentes intermedios" de algo "más grande". | El servidor posee la mayor lógica de negocio, mientras que el cliente se limita a presentar los datos. Por ejemplo: Wikipedia. |
| Cliente Activo | El cliente es quien posee la mayor lógica de negocio. El servidor limita su funcionalidad a la persistencia. | La lógica de negocio está distribuida en ambos componentes. Por ejemplo: Linkedyn. |

### Protocolo HTTP

HTTP utiliza el puerto 80, mientras que HTTPS utiliza el puerto 443.

Verbos del protocolo HTTP V1.1
* GET: Solicita un recurso sin modificarlo. Por ejemplo: Cargar página web.
* POST: Envía datos al servidor para que sean procesados. Por ejemplo: Enviar un formulario.
* PUT: Actualiza o crea un recurso en el servidor. Maneja de forma total de un recurso.
* PATCH: Modifica parcialmente un recurso.
* DELETE: Elimina un recurso del servidor.

Códigos de estado:
* 1xx: Respuestas informativas. Indica que la petición ha sido recibida y se está procesando.
* 2xx: Respuestas correctas. Indica que la petición ha sido procesada correctamente.
  * 200: La solicitud fue exitosa.
  * 201: El servidor da de alta correctamente un recurso.
  * 204: Solicitud exitosa, pero el cliente no necesita ver una página nueva.
* 3xx: Respuestas de redirección. Indica que el cliente necesita realizar más acciones para finalizar la petición.
* 4xx: Errores causados por el cliente. Indica que ha habido un error en el procesado de la petición a causa de que el cliente ha hecho lago mal.
  * 400: No se puede procesar la solicitud o existe un problema con la solicitud del cliente.
  * 401: Al cliente le falta las autenticaciones necesarias para realizar la solicitud.
  * 403: El usuario realiza una solicitud de la cual no cuenta con los permisos necesarios.
  * 404: El recurso solicitado no existe.
  * 409: Indica conflictos.
* 5xx: Errores causados por el servidor. Indica que ha habido un error en el procesado de la petición a causa de un fallo en el servidor.

### API y API REST

> Una API (Interfaz de Programación de Aplicaciones) es un conjunto de herramientas, definiciones y protocolos que se usan para diseñar e integrar aplicaciones.

Tipos de API
* Arquitectura
  * Local: Proporciona servicios de Sistema Operativo, Bibliotecas o Middleware.
  * Web: Proporciona servicio entre componentes distribuidos y en general sobre el protocolo HTTP.
  * Programa: Proporciona la posibilidad de ejecución de código de forma remota entre componentes distribuidos (RPC).
* Alcance
  * Public (Pública)
  * Partner (Para terceros)
  * Private (Privado)
