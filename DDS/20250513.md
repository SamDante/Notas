## Notas de la clase

### Integración de sistemas

* Interface Entrante: Es aquel contrato que define qué datos necesita un componente para poder realizar una tarea / ejecutar una funcionalidad.
* Interface Saliente: Es aquel contrato que define qué datos necesita el componente al cual uno se quiere integrar para poder realizar una tarea / ejecutar una funcionalidad.

* Entorno de ejecución: Es un contexto que refiere a hardware y software donde se ejecuta una aplicación. Al integrar componentes se debe tener en cuenta dónde están ubicados:
  * En el mismo sistema y en el mismo entorno
  * En el mismo sistema, pero con distinto entorno.
  * En distintos sistemas y en distintos entornos.
  
* Testing con integraciones: Si se tiene que testear un componente propio con una integración con otro que no está en el mismo entorno se debe considerar:
  * La interfaz saliente debe estar mockeada, porque sino se estaría llamando al componente en cada test.

### Cuestiones de sincronismo

> Cuando un componente A hace uso de alguna funcionalidad expuesta por el componente B y *se queda esperando la respuesta*, entonces A está integrandose de forma *sincrónica* con B para la utilización de dicha funcionalidad.

> Cuando un componente A hace uso de alguna funcionalidad expuesta por el componente B y *no se queda esperando la respuesta*, entonces A está integrándose de forma *asincrónica* con B para la utilización de dicha funcionalidad.

* El componente A puede no quedarse esperando la respuesta de B por alguno de estos motivos:
  * No necesita el resultado / respuesta para seguir con la ejecución.
  * No le interesa el resultado / respuesta.

### Cronjobs

CronJobs = CronTab

> Un CronJob es una tarea programada que se ejecuta de manera periódica y automática en Sistemas Unix-Like. Se utiliza para automatizar comandos o scripts en intervalos de tiempo definidos por el usuario. Sirven para el proceso de forma asíncrona.

* Se define en una línea dentro de un archivos llamado "crontab" y tiene la estructura: (Minuto [0 - 59]) (Hora [0 - 13]) (Día del mes [1 - 31]) (Mes [1 - 12]) (Día de la semana [0 - 7]) comando. Ingresar un "*" en alguno de estos parámetros indica que se lo haga durante "todo" ese rango.
