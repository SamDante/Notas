## Notas de la clase

Casa Segura: Por disponibilidad (el sistema debe funcionar aún sin Internet) y performance (en caso de alguna amenaza / accidente el sistema debe atenderlo lo más pronto posible) se decide que el sistema tenga una instancia desplegada en cada casa, para centralizar las actualizaciones se debe utilizar algun sistema externo.

Se debe crear una clase Hogar, ya que éste conlleva atributos que se deben poder guardar, configurar y mostrar por pantalla.

Se separa la ocurrencia del evento para la clase hogar, con la detección de un evento como tal.

Se modelan las camaras de seguridad y sensores de movimiento como gemelos digitales (representan a todos los dispositivos físicos utilizados). Las cuales heredan de la clase ReceptorRoboX que a su vez hereda de la clase abstracta ReceptorRobo.

La clase abstracta ReceptorRobo contiene los métodos protegidos para decodificar los datos recibidos, analizar los datos del robo para obtener el grado de riesgo, y verificar la existencia de un robo segun el grado de riesgo.

Patrón Command: Se establece una interfaz para comandar a las clases que lo implementan, sus clases se nomenclan como métodos
