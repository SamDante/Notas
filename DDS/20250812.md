## Notas de la clase

Persistencia: Refiere a que el estado de un sistema sobrevive más allá del proceso que lo creó. En la práctica, esto se logra almacenando dicho estado en algún (o algunos) medio persistente, como una base de datos o archivos. Existen varias estrategias de persistencia de datos de un dispositivo:

* Persistencia en memoria (RAM): Es utilizado mayormente para realizar test en los sistemas.
* Persistencia en archivos: Involucra guardar el estado de un sistema en uno o varios archivos, para que luego se puedan acceder y retomar el estado del sistema tal como estaba.
* Prevalencia: Almacena el estado del sistema en la memoria principal, pero que genera "snapshots" a disco para evitar la pérdida completa de datos.
* Ortogonal: Se implementa como una propiedad intrínseca de su entorno de ejecución, por lo que no se requieren de acciones específicas para poder guardar o recuperar datos.
* Persistencia en base de datos: El estado del sistema es persistido en uno (o varias) bases de datos. Otorgan mecanismos para recuperarse en caso de fallos, además de brindar reglas para resguardar la integridad de los datos.
  * Bases de datos de Objetos: Está atada a un leguaje de programación en particular, ya que cada lenguaje tiene una forma de representar los objetos particular.

### Base de Datos Relacional

Clave primaria (Primary Key): Es un atributo que identifica de manera unívoca a cada fila de la tabla. Existen de 3 tipos.

* Clave primaria natural: Son campos únicos por definición.
* Clave primaria subrogada: Son claves inventadas para funcionar como claves primarias. Ej: Id numérico.
* Clave primaria compuesta: Se toman dos o más campos para que diferencien al registro del resto. Ej: DNI y Pasaporte.

Índices: Es una estructura de datos que mejora la velocidad de las operaciones, por medio de un identificador único de cada fila de una tabla.

Clave foránea (Foreign Key): Es un grupo de una o más columnas en una tabla que referencian a la clave primaria de otra tabla. Evita la duplicidad de datos. Una FK puede ser parte de una PK.

Tanto las PK como las FK son índices.

Integridad referencial: Es una propiedad que establece que la clave externa de una tabla de referencia siempre debe corresponder a una fila válida de la tabla a la que se haga referencia.

| Tipo de dato | Descripción |
| :-: | :-: |
| VARCHAR | Cadenas de caracteres compuestas por letras, números y caracteres especiales. Tiene una longitud ajustable hasta 255 caracteres. |
| INTEGER | Números enteros naturales |
| DOUBLE | Números con punto flotante (decimales) |
| DATETIME | Formato para manejo de fechas |

Constraints: Son restricciones al modelo utilizadas para limitar el tipo de dato que puede ingresarse en una tabla. Son especificados cuando se crea la tabla o pueden ser especificados luego con una actualización. Estos pueden ser: Not Null (no acepta valores nulos), Unique (no acepta valores repetidos), Check / enum (verifica que los valores cumplan determinada condición), Primary Key, Foreign Key.

Cardinalidad: Es la cantidad máxima de relaciones que tiene un registro de una entidad con respecto a la otra tabla.

Modalidad: Es la cantidad mínima de relaciones que tiene un registro de una entidad con respecto a la otra tabla.
